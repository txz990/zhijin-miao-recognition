# 仓库与论文差异说明

本文档用于说明当前代码仓库与论文实验描述之间的边界，避免把“研究结果”与“仓库默认能力”混为一谈。

## 1. 数据规模差异

论文与方案文档中出现过多组数据规模描述，例如：

- `50` 个词、`200` 条语音
- `200` 个词、`490` 条样本

当前仓库默认初始化脚本 `src/main/resources/schema.sql` 只提供：

- `5` 名发音人示例数据
- `3` 条词汇示例数据
- 不内置完整 `audio_samples`
- 不内置完整 `templates`

这意味着：

- 仓库可以直接启动并验证系统结构
- 但默认不具备论文中的完整实验数据规模

## 2. 性能指标差异

论文或方案文档中的下列指标：

- `94.7%` 准确率
- `450ms` 平均处理时间
- `RTF 0.53`

属于研究实验结果，不应理解为当前仓库在默认示例数据上的即时运行结果。

当前仓库：

- 没有内置完整实验语料
- 没有固定实验划分脚本
- 没有完整评测脚本

因此这些数字目前不能视为“开箱即复现”的仓库指标。

## 3. 实验代码缺失项

论文中提到但当前仓库未完整提供的实验级能力包括：

- HMM 对比实验
- 噪声鲁棒性实验
- 说话人相关 / 说话人无关实验
- 模板数量对比实验
- 论文级交叉验证流程

这些内容更接近“实验与评测工程”，而不是系统主服务本身。

## 3.1 接口路径兼容说明

仓库中曾出现过两套识别接口路径：

- 旧写法：`/api/v1/recognition/...`
- 当前统一写法：`/recognition/...`

当前后端已经同时兼容这两套前缀，避免旧页面或旧文档直接失效。

后续新增页面、文档和脚本建议统一使用：

- `/recognition/recognize`
- `/recognition/statistics`
- `/recognition/history`

## 4. 当前仓库已实现的能力

当前代码仓库已经具备以下基础能力：

- 音频上传识别
- VAD 端点检测
- 39 维 MFCC 特征提取
- DTW 模板匹配
- 拒识机制
- 识别历史记录
- 基础模板重建接口

## 5. 如果要补齐论文级复现

建议按下面顺序继续扩展：

1. 导入统一格式的完整语料和词表
2. 增加训练/测试划分与交叉验证脚本
3. 增加模板数量对比实验脚本
4. 增加噪声注入与鲁棒性评测脚本
5. 增加说话人相关 / 无关评测脚本
6. 如确有需要，再补 HMM 对比基线

## 6. 当前原则

本仓库后续优先遵循以下原则：

- 代码里只保留当前可成立的功能说明
- 论文实验结果单独标注，不冒充仓库默认结果
- 演示数据与完整实验语料严格区分
